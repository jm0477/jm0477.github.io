Linux内核爆glibc模块漏洞-检测及修补方法

发表于 2015-01-28

今天突然看到新闻glibc被一家code审计公司爆出溢出漏洞

内容如下：

GNU glibc 标准库的 gethostbyname
函数爆出缓冲区溢出漏洞，漏洞编号：CVE-2015-0235，漏洞详情请看这里。

该漏洞使黑客能够在不知道系统的id和密码的情况下远程控制系统。通过glibc的gethostbyname函数触发。这个函数几乎被用于所有联网的Linux电脑（当其被另一个联网的计算机使用/etc/hosts等文件访问时，或者更常见的通过DNS来解析一个互联网域名）。

利用该漏洞，所有攻击者可以通过对一个DNS解析应用使用一个无效的主机名参数来引发缓冲区溢出。然后远程攻击者就能够获得用户运行DNS权限来执行任意的代码，简单的来说，一旦攻击者利用了GHOST那么他们就可能掌控你的系统。

Glibc 是提供系统调用和基本函数的 C 库，比如open, malloc, printf等等。所有动态连接的程序都要用到它。

赶紧用下面的方法编译查看自己的内核是否脆弱吧：

编译一下内容

#include #include #include #include #include #include #define CANARY "in_the_coal_mine"struct {char buffer[1024];char canary[sizeof(CANARY)];} temp = { "buffer", CANARY };int main(void) {struct hostent resbuf;struct hostent *result;int herrno;int retval;/*** strlen (name) = size_needed - sizeof (*host_addr) - sizeof (*h_addr_ptrs) - 1; ***/size_t len = sizeof(temp.buffer) - 16*sizeof(unsigned char) - 2*sizeof(char *) - 1;char name[sizeof(temp.buffer)];memset(name, '0', len);name[len] = '\0';retval = gethostbyname_r(name, &resbuf;, temp.buffer, sizeof(temp.buffer), &result;, &herrno;);if (strcmp(temp.canary, CANARY) != 0) {puts("vulnerable");exit(EXIT_SUCCESS);}if (retval == ERANGE) {puts("not vulnerable");exit(EXIT_SUCCESS);}puts("should not happen");exit(EXIT_FAILURE);}

执行： 

gcc gistfile1.c -o CVE-2015-0235./CVE-2015-0235

如果是Vulnerable， 赶紧更新吧~~

修复方案：

升级glibc库

RHEL/CentOS yum update glibc

UUbuntu: sudo apt-get update; sudo apt-get install libc6

我的服务器升级前 glibc版本为：

glibc-2.12-1.149.el6.x86_64

升级后：

glibc-2.12-1.149.el6_6.5.x86_64

在执行下：

gcc gistfile1.c -o CVE-2015-0235./CVE-2015-0235

not vulnerable

搞定，，真心吓一跳。。这漏洞实在太nb了。。。

使用powershell发送邮件

powershell内置一个cmdlet名字叫Send-mailMessage

Send-MailMessage -to "xxx@xxx.com" -from "fake@abc.com" -Subject "test from smtp"-Body $l -SmtpServer yoursmtp.servername

这里-body 只能接受string， 如果你的body变量$l 不是string

powershell的远程管理前提

发表于 2014-09-04

想用本机来控制远程的机器

远程机器的设置，主要是，在管理员权限下的PowerShell控制台中执行

Enable-PSRemoting -force

本机和远程机器如果在同一个域中，就只需要做上面的设置了。

如果本机和远程机器不在同一个域，则可以通过设置域信任达到相同的目标。

如果本机和远程机器都不在域中，则需要将远程机器加入到本机的TrustedHosts

set-item wsman:localhost\client\trustedhosts -value <远程机器名>

然后就可以建立远程会话了

New-PSSession -ComputerName <远程机器名>

在通过Invoke-Command就可以让在远程机器上运行对应的脚本了

用powershell检测tcp端口连通性

发表于 2014-09-02

powershell中没有内置cmdlet来做tcp的测试

所以需要调用.net类库

实现代码如下：

 function Check-Port ($dest,$port)
{
    $tcp= New-Object net.sockets.tcpclient
    $tcp.Connect($dest,$port) 
    If ($? -ne "True")
    {return $false}
    Else 
    {
    $tcp.Close()
    return $true
    }
}

我没有使用try去抓异常，因为我只需要返回结果来判断下一步的操作。

还有注意，在c#养成的调用习惯要改。刚才调用的时候莫名出现n多错误，什么unknow host之类的

正确调用：

Check-Port -dest 127.0.0.1 -port 80

Could not load file or assembly (HRESULT 0x80131515)

发表于 2014-08-28

下载的附件里面包含dll文件，会自动被系统block，无法使用

必须手动点开属性，点unblock

彻底解决这个问题，去组策略里面启动一个：

  1. 1.gpedit.msc

  2. 2. User Configuration > Administrative Templates > Windows Components > Attachment Manager

  3. 3. Enable: Do not preserve zone information in file attachments

再下载的附件，就不会被系统自动锁住，导致标题这个错误了。
