CentOS6.5搭建LNMP

发表于 2015-05-21

CentOS6.5搭建LNMP

1：查看环境：

[root@10-4-14-168 html]# cat /etc/redhat-release
CentOS release 6.5 (Final)

2：防火墙配置，开启80/3306端口

vi /etc/sysconfig/iptables
-A INPUT -m state --state NEW -m tcp -p tcp --dport 80 -j ACCEPT #允许80端口通过防火墙
-A INPUT -m state --state NEW -m tcp -p tcp --dport 3306 -j ACCEPT #允许3306端口通过防火墙

ps： 这两条必须添加在默认22端口这条规则下面，添加后应该是如下状态：

# Firewall configuration written by system-config-firewall
# Manual customization of this file is not recommended.
*filter
:INPUT ACCEPT [0:0]
:FORWARD ACCEPT [0:0]
:OUTPUT ACCEPT [0:0]
-A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
-A INPUT -p icmp -j ACCEPT
-A INPUT -i lo -j ACCEPT
-A INPUT -m state --state NEW -m tcp -p tcp --dport 22 -j ACCEPT
-A INPUT -m state --state NEW -m tcp -p tcp --dport 80 -j ACCEPT
-A INPUT -m state --state NEW -m tcp -p tcp --dport 3306 -j ACCEPT
-A INPUT -j REJECT --reject-with icmp-host-prohibited
-A FORWARD -j REJECT --reject-with icmp-host-prohibited
COMMIT

最后重启防火墙使配置生效
/etc/init.d/iptables restart

关闭SELINUX

vi/etc/selinux/config, 注释如下两条命令

#SELINUX=enforcing
#SELINUXTYPE=targeted

3：配置CentOS 6.0 第三方yum源（CentOS默认的标准源里没有nginx软件包）

wget http://www.atomicorp.com/installers/atomic
sh ./atomic
yum check-update

4：安装开发包和库文件

yum -y install ntp make openssl openssl-devel pcre pcre-devel libpng libpng-devel libjpeg-6b libjpeg-devel-6b freetype freetype-devel gd gd-devel zlib zlib-devel gcc gcc-c++ libXpm libXpm-devel ncurses ncurses-devel libmcrypt libmcrypt-devel libxml2 libxml2-devel imake autoconf automake screen sysstat compat-libstdc++-33 curl curl-devel

5：卸载已安装的apache、mysql、php

yum remove httpd
yum remove mysql
yum remove php

6：安装nginx

yum install nginx
service nginx start
chkconfig --levels 235 nginx on

//设2、3、5级别开机启动

7：安装mysql

yum install mysql mysql-server mysql-devel
service mysqld start
chkconfig --levels 235 mysqld on

登陆MySQL删除空用户，修改root密码

mysql>select user,host,password from mysql.user;

mysql>drop user ''@localhost;

mysql>update mysql.user set password = PASSWORD('*********') where user='root';

mysql>flush privileges;
　　

8：安装php

yum install php lighttpd-fastcgi php-cli php-mysql php-gd php-imap php-ldap
php-odbc php-pear php-xml php-xmlrpc php-mbstring php-mcrypt php-mssql php-snmp php-soap

//安装php和所需组件使PHP支持MySQL、FastCGI模式

yum install  php-tidy php-common php-devel php-fpm php-mysql
service php-fpm start
chkconfig --levels 235 php-fpm on

9：配置nginx支持php

mv /etc/nginx/nginx.conf /etc/nginx/nginx.confbak

//将配置文件改为备份文件

cp /etc/nginx/nginx.conf.default /etc/nginx/nginx.conf

//由于原配置文件要自己去写因此可以使用默认的配置文件作为配置文件

//修改nginx配置文件，添加fastcgi支持

vi /etc/nginx/nginx.conf

index index.php index.html index.htm;
//加入index.php

location ~ \.php$ {
            root           /usr/share/nginx/html;
            fastcgi_pass   127.0.0.1:9000;
            fastcgi_index  index.php;
            fastcgi_param  SCRIPT_FILENAME  /usr/share/nginx/html$fastcgi_script_name;
            include        fastcgi_params;
        }
//将以上代码注释去掉，并修改成nginx默认路径

取消FastCGI server部分location的注释,并要注意fastcgi_param行的参数,/scripts$fastcgi_script_name改为$document_root$fastcgi_script_name,或者使用绝对路径，要不然浏览器访问，不能解析，会显示“ File not found.

10：配置php

//编辑文件php.ini，在文件末尾添加cgi.fix_pathinfo = 1
[root@CentOS ~]# vi /etc/php.ini

11：重启nginx php-fpm

service nginx restart
service php-fpm restart
　　

12：建立info.php文件

vi /usr/share/nginx/html/info.php

13：测试nginx是否解析php

本地浏览器输入：192.168.1.105/info.php

显示php界面 环境搭建成功

用squid做socks5的代理实践

发表于 2015-04-29

原理是用socks5代理来上网，用polipo来转换socks5到http，然后用squid在多个http代理见做负载均衡

记录下过程：

1. 先安装shadowsoks

sudo apt-get install python-pip
sudo apt-get install python-gevent #可以提高shadowsock性能
sudo pip install shadowsocks

然后创建个config.json文件，内容如下：

{
"server":"server_ip_address",
"server_port":8388,
"local_address":"127.0.0.1",
"local_port":local_port,
"password":"your_passwd",
"timeout":600,
"method":"aes-256-cfb",
"fast_open":false,
"workers":1
}

然后运行 sslocal -c /etc/shadowsocks/config.json即可运行

如果需要不受当前session限制，可以使用nohup命令后台运行：

nohup sslocal -c /etc/shadowsocks/config.json > /dev/null 2>&1 &

2. 防火墙设置(如果开着)

-A INPUT -m state --state NEW -m tcp -p tcp --dport 8388 -j ACCEPT
service iptables restart

3. 安装polipo

polipo是一个轻量级的http代理，并且能将socks5代理转换成http，我们这里只用socks转换功能。

sudo apt-get install polipo

配置polipo:

编辑/etc/polipo/config， 内容如下：

logSyslog = true
logFile = /var/log/polipo/polipo.log
proxyAddress = "0.0.0.0"
proxyPort = 8123 #将要使用的转换成http的端口
socksParentProxy = "0.0.0.0:8388"  #这里的端口为上级socks5代理的端口
socksProxyType = socks5

开启polipo服务：

service polipo restart

ok，现在已经有转换好可用的http代理，下面来配置squid来做负载均衡：

4. squid安装

sudo apt-get install squid3

squid配置：

编辑 /etc/squid3/squid.conf 

visible_hostname balancer
http_port 3128
cache_peer 127.0.0.1 parent 8123 0 no-query no-digest round-robin weight=1 name=shadowsocks
http_access allow all
never_direct allow all

这里只是简单配置所有人都可以访问，以后可以使用acl严格控制访问

!!!要注意这句never_direct allow all

这句的意思是不直接走squid代理出去，而使用我们配置好的parent代理出去

搞了好久，也无法科学上网，就是因为这句 never_direct allow all

好了，打开squid,

service squid start

写的很简陋，但是思路对，慢慢研究吧。

python装饰类

发表于 2015-04-02

最近在看python的基础教程，看到decorator这里有点乱，，，记录一下，以后查阅起来也方便点。

就是用decorator方法来装饰类，达到封装类的功能，添加类的新功能

def decorator(aClass):    
            class newClass:
                    def __init__(self, age):
                        self.total_display   = 0
                        self.wrapped         = aClass(age)
                    def display(self):
                        self.total_display += 1
                        print("total display", self.total_display)
                        self.wrapped.display()
            return newClass

@decorator
class Bird:    def __init__(self, age):
        self.age = age
        def display(self):        
        print("My age is",self.age)

eagleLord = Bird(5)
    for i in range(3):
        eagleLord.display()

在decorator中，我们返回了一个新类newClass。在新类中，我们记录了原来类生成的对象（self.wrapped），并附加了新的属性total_display，用于记录调用display的次数。我们也同时更改了display方法。

self.wrapped的值其实是一个可调用的函数，self.wrapped.display()则实际上是对decorator封装类aClass(new)的display方法的调用。

在decorator中，self.wrapped类及display方法都被return，生成新的class aClass

定义好装饰器后，我们就可以通过@语法使用了，@紧接着的class Bird，实际上是将class Bird
传递给decorator，并将decorator返回的新的可调用对象赋给原来的函数名

在实例化Bird类的eagleLord对象中，调用对象的display() ，相当于

eagleLord= decorator(Bird)

我们知道，Python中的变量名和对象是分离的。变量名可以指向任意一个对象。从本质上，装饰器起到的就是这样一个重新指向变量名的作用(name
binding)，让同一个变量名指向一个新返回的可调用对象，从而达到修改可调用对象的目的。

与加工函数类似，我们可以使用装饰器加工类的方法。

如果我们有其他的类似函数，我们可以继续调用decorator来修饰函数，而不用重复修改函数或者增加新的封装。这样，我们就提高了程序的可重复利用性，并增加了程序的可读性。

Windows Server 2012从评估版转成正式版

发表于 2015-03-28

公司某台测试服务器最初安装了Evaluation版Windows Server
2012,测试完成通过KMS激活时提示非核心系统无法激活。这时可以通过将评估版转成正式版后再通过KMS激活。方法如下：

1、以管理员权限运行cmd，输入DISM /online /Get-CurrentEdition，查看当前系统版本，或通过winver命令图形界面查看。

2、输入DISM /online /Set-Edition: /ProductKey:XXXXX-XXXXX-XXXXX-XXXXX-XXXXX
/AcceptEula

为当前可以转换的版本，ProductKey为产品KEY。

3、如需把标准版再升级至数据中心或企业版，可以先输入dism /online /get-
targeteditions，查看当前系统可以升级的目标版本（如下图），再通过第二步命令升级至较高版本。

4、升级完成后，重新启动系统，系统已变成正式版。再通过KMS服务器激活，激活成功。

PS：有关DISM命令的具体用法，请参考http://technet.microsoft.com/zh- cn/library/dd744380(v=ws.10).aspx

DISM /online /Set-Edition:ServerStandard /ProductKey:XC9B7-NBPP2-83J2H-RHMBY-
92BT4 /AcceptEula
