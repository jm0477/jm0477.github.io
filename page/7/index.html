powershell的远程管理前提

发表于 2014-09-04

想用本机来控制远程的机器

远程机器的设置，主要是，在管理员权限下的PowerShell控制台中执行

Enable-PSRemoting -force

本机和远程机器如果在同一个域中，就只需要做上面的设置了。

如果本机和远程机器不在同一个域，则可以通过设置域信任达到相同的目标。

如果本机和远程机器都不在域中，则需要将远程机器加入到本机的TrustedHosts

set-item wsman:localhost\client\trustedhosts -value <远程机器名>

然后就可以建立远程会话了

New-PSSession -ComputerName <远程机器名>

在通过Invoke-Command就可以让在远程机器上运行对应的脚本了

powershell中没有内置cmdlet来做tcp的测试

所以需要调用.net类库

实现代码如下：

 function Check-Port ($dest,$port)
{
    $tcp= New-Object net.sockets.tcpclient
    $tcp.Connect($dest,$port) 
    If ($? -ne "True")
    {return $false}
    Else 
    {
    $tcp.Close()
    return $true
    }
}

我没有使用try去抓异常，因为我只需要返回结果来判断下一步的操作。

还有注意，在c#养成的调用习惯要改。刚才调用的时候莫名出现n多错误，什么unknow host之类的

正确调用：

Check-Port -dest 127.0.0.1 -port 80

powershell调用selenium模拟浏览器操作

发表于 2014-08-28

什么是Selenium

Selenium是一款著名的web应用程序测试工具，它能通过在浏览器中模拟用户的动作来完成测试，其api支持java，c#，python，ruby，php，perl，javascript这些主流编程语言和脚本语言。Selenium还支持IE，火狐，和chrome等主流浏览器。

PowerShell 如何直接调用Selenium

PowerShell直接调用Selenium，其实类似PowerShell调用C#方法。以IE浏览器为例，至少需要具备两个文件：

  1. IEDriverServer.exe 启动IE代理，模拟用户操作

  2. WebDriver.dll 暴露API给用户，对浏览器进行控制

纯PowerShell调用Selenium的例子

为什么要使用Selenium

  1. 看了上面的调用，有的哥们要开始质疑了这里面主要的方法是：找element。和InternetExplorer.Application中的Html Document类中的关键方法类似啊，byid，byName，byClass，ByTag。这么想，没错。

  2. 但是经本人测试，如果web页面稍显复杂，InternetExplorer.Application中的除了getElementById可以勉强接受，其它的方法慢的一塌糊涂。而Selenium却表现非常良好。

  3. 另外Selenium支持css selector查找结点，这无疑是最给力的了，让jquery也投入战斗。

  4. 最后一点当然是Selenium对于浏览器的兼容性优势了。

powershell调用.net对象发送邮件

发表于 2014-08-27

一个powershell脚本使用smtp协议发送邮件的function，带抛STMP异常及异常提示

function send-mail($to,$subject,$body)
{
    try
    {
        $SmtpServer="smtp.qq.com"
        $smtpuser="username"
        $smtppassword="password"
        $mail=New-Object System.Net.Mail.MailMessage
        $mailaddress="277347949@qq.com"
        $mailtoAddress=$to
        $mail.From= New-Object System.Net.Mail.MailAddress($mailaddress)
        $mail.To.Add($mailtoAddress)
        $mail.Subject=$subject
        $mail.Priority="High"
        $mail.Body=$body
        $smtp=New-Object System.Net.Mail.SmtpClient -ArgumentList $SmtpServer
        $smtp.Credentials=New-Object System.Net.NetworkCredential -ArgumentList $smtpuser,$smtppassword
        $smtp.Send($mail) 
    }
    catch [System.Net.Mail.SmtpException],[System.Net.Mail.SmtpFailedRecipientException]
    {
        write-host "SMTP server is down or unavailable! please check your SMTP settings"
    }
    catch
    {
        write-host 'the recipient is not existed, please check your "-to" value'
    }
}

用poweshell定期检查openvpn服务状态并自动修复

发表于 2014-08-25

有一台本地服务器，要连接国外的vpn服务器，但是国内网络环境你懂的。

连接很不稳定，据观察，基本上2-3天就会断开连接，但是openvpn重试无法连接x次后

就不会继续尝试，而报出一个messagebox，就完事儿了。

可是vpn不就挂在那，木有人知道了么！

本着解决这个vpn链接问题，想法是：

  1. 用脚本定期运行，检查vpn连接状态是否正常

  2. 检测到vpn连接不正常，要重启vpn连接服务

所以脚本调试几次，ok了贴在下面， task scheduler
配置很方便，GUI下更简单，选好你需要的执行间隔，然后在Action–>Scripts/Program下面写 powershell

在Add argument里面填写 powershell.ps1文件的路径即可！

脚本是简单的检测log的方法，大家可以来参考：

#通过windows的task shedule，定期运行获取当前log信息中是否包含链接失败，自动重连openvpn
$logs=Get-Content 'C:\Program Files (x86)\OpenVPN\log\Openvpn.log'
$i=1
foreach($log in $logs)
{
    if($log.Contains("Connection refused"))
    {
        $i=$i+1
    }


}
if($i -gt 5)
{
    #停止vpn尝试，关闭进程，重启vpn连接
    get-process | where{$_.processname -like "*openvpn*"} | stop-process
    sleep 5
    & 'C:\Program Files (x86)\OpenVPN\bin\openvpn-gui-1.0.3.exe' --connect openvpn.ovpn

}
else
{
    #检查一切正常，结束程序
}

powershell脚本练习-获取批量计算机运行状态

发表于 2014-08-22

服务器每个月定期打补丁，打补丁必然要重启，但是重启后，是否能正常运行是个未知数

所以我试着写了这个脚本，可以批量获取所有你需要ping的机器，帮助你查看这个列表里所有机器的连接状态

不失为一种偷懒，减少工作量的tips

用这个脚本，只需要将需要检测的server列表更新到指定的txt文件，剩下就是等待脚本慢慢给你执行吧

$computers= Get-Content C:\Users\oe-jingming\Desktop\computers.txt
do
{
    $cons=Test-Connection $computers -count 1
    for ($i = 0; $i -lt $cons.length; $i++)
    { 
        if($cons[$i].statuscode -ne 0)
        {
           write-host $cons[$i].protocoladdress "is down" 
        }
        else
        {
            write-host $cons[$i].protocoladdress "is up and running"
        }
        $sum=$cons[$i].statuscode+ $cons[$i+1].statuscode
    }
    if ($sum -eq 0)
    {
        write-host "All server is up and running"
    }
    else
    {
        write-host "Server is not up"
    }

}
while ($cons.statuscode -eq 0)

另外一个版本，没有循环，只有批量输出

<#
.Synopsis
   This script is used to ping a list of servers from cmd prompt
.DESCRIPTION
   This script is written to accept a list of servers either from pipeline or from input and write the output to the screen
.EXAMPLE
   "baidu.com","google.com" | ping-Host 
.EXAMPLE
   Ping-Host -Host "localhost","baidu.com" | Format-Table -autosize
.EXAMPLE
PS C:\Users\Administrator\Desktop> gc .\servers.txt | Ping-Host | Sort-Object Status

Status ReponseTime                           Hostname                             IPAddress
------ -----------                           --------                             ---------
False Offline                               offline                              Offline
False Offline                               notonline                            Offline
False Offline                               microsoft.com                        Offline
 True 387                                   mindtree.com                         209.61.153.180
 True 0                                     localhost                            127.0.0.1
 True 3                                     baidu.com                            123.125.114.144
 True 190                                   google.com                           173.194.127.41
#>
function Ping-Host()
{
    [CmdletBinding()]

    Param
    (
        # Param1 help description
        [Parameter(Mandatory=$true,
                   ValueFromPipeline=$true,
                   ValueFromPipelineByPropertyName=$true)]
        [string[]]$server
    )

    Begin
    {
    $obj=@()
    }
    Process
    {
        Foreach($b in $server)
        {
            $R = Get-WmiObject -Class Win32_PingStatus -Filter "Address='$b'"
            Write-Progress -Activity "Pinging list of servers" -Status "Please wait" -CurrentOperation "$b in progress"
                If($r.statuscode -eq 0)
                {
                    $property = @{'IPAddress'= $r.IPV4Address ;`
                                  'Hostname'=$b; `
                                  'ReponseTime'=$r.ResponseTime;`
                                  'Status' = $True}
                    $object = New-Object -TypeName Psobject -Property $property
                }
                Else
                {
                    $property = @{'IPAddress'= "Offline" ;`
                                  'Hostname'=$b; `
                                  'ReponseTime'= "Offline";`
                                  'Status' = $false}
                    $object = New-Object -TypeName Psobject -Property $property
                }
            $obj+= $object
            }
    }
    End
    {
    $obj
    }
}

powershell安装注册表信息

今天服务器前期部署都结束了，准备安装SCOM Agent放入监控系统

SCOM 要通过discovery 特定的registry
key来判断服务器的类型，从而target到特定monitor和rule来完成监控和产生alert
