Django学习08 django通用视图- 制作友好的模板Context

发表于 2016-04-08

通用视图例子:

from django.views.generic import list_detail

publisher_info = {

‘queryset’: Publisher.objects.all()

#lesson1/publisher_info.html 如果没指明，会自动寻找此模板

‘template’: ‘publisher_info.html’

}

urlpatterns = patterns(‘’,

(r’^publisher/$’, list_detail.object_list, {‘publisher_info’: publisher_info})

)

=================================================================================

模板中调用object_list时，不友好，因为我们不知道object_list里面是什么

我们就可以将publisher_info字典定义一个

‘template_object_name’ : ‘publisher_list’

指定它叫做publisher_list

然后再模板中就可以优雅的调用:

这样就避免了直接使用list_detail下的object_list方法，而改名成更友好的publisher_list直接在模板中调用，很直观

当然有时候一个页面有多个数据源，我们还需要添加额外的处理器

比如说这次处理需要调用Publisher模型及Book模型两种数据

就可以在

publisher_info字典中定义额外：

def find_book():

return Book.objects.all()

publisher_info = {

‘queryset’: Publisher.objects.all()

‘template_object_name’ : ‘publisher_list’

‘template’: ‘publisher_info.html’

‘extra_context’ : {‘book_info’: Book.objects.all()} #定义一个新的处理器

}

但是这样写，有一个小bug， 就是这段程序，只有在启动这个app的时候，在相应的view中运行一次， 以后再次调用，如果数据在此期间修改过。
不会返回更新后的数据.

所以这里要写一个函数引用, 满足通过view渲染的时候才会调用

def find_book():

return Book.objects.all()

publisher_info = {

‘queryset’: Publisher.objects.all()

‘template_object_name’ : ‘publisher_list’

‘template’: ‘publisher_info.html’

‘extra_context’ : {‘book_info’: find_book()} #动态加载Book.bjects.all()

}

函数包装- 来处理复杂的数据过滤

比如说， 想查询制定的publisher下面的所有书

urlpatterns = patterns(‘’,

(r’^books/(\w+)/$’, books_by_publisher’)

)

views.py

from django.shortcuts import get_object_or_404

from lesson1.models import Publisher

def books_by_publisher(request, name):

publisher = get_object_or_404(Publisher, name = name)

return list_detail.object_list(request, queryset =
books.objects.filter(Publisher = publisher),

template_objects_name = ‘book’,

extra_context = {‘publisher’: publisher})

通过 publisher查到指定的publisher, 然后传入queryset来查询指定publisher的书籍，

然后指定了一个友好的template_object_name= ‘book’

然后写了一个额外的context，将publisher和book一并送入list_details.object_list返回

from django.template import Http404

from django.views.generic.simple import direct_to_template

def abouts(request, page):

try:

return direct_to_template(request, template = ‘about/$s.html’, % page)

except TemplateDoesNotExisted:

Django学习01 URL初级配置

发表于 2016-04-06

Django学习 URL初级配置

常见的URL传值分两类：

1. www.881027.com/?id=123?name=richie

涉及到的模块：

views.py：

需要配置一个time的view函数，来接受request，使用request.GET.get(‘id’)的形式来依次接受从左至右的参数值，多个参数之间用&连接，例如www.881027.com/time/?id=123?name=richie

def time(request):

t = loader.get_template(“time.html”)

id = request.GET.get(‘id’)

name = request.GET.get(‘name’)

c = Context({“today”:datetime.datetime.now(),”id”:id,”name”:name},)

return HttpResponse(t.render(c))

urls.py:

主urls配置文件，使用include方法，将匹配到blog/开头的全部转向当前app目录下的blog.urls.py文件

from django.conf.urls import patterns, include,url

urlpatterns = patterns(‘blog.views’,

url(r’blog/‘, include(‘blog.urls’))

)

接下来，在当前app目录下，创建urls.py

from django.conf.urls import patterns, include,url

urlpatterns = patterns(‘blog.views’,

url(r’^time/$’, ‘time’),

)

time.html:

此为模板页

在段输出相应字段即可

  • Today is {{ today | date:"Y-m-d" }}

  • ID is {{ id }}

  • Name is {{ name }}

2. www.881027.com/time/123/richie

这种方式用户方式比较友好，但是在urls.py里要配置好需要接受的正则表达式，略复杂：

views.py：

def time(request,p1,p2):

t = loader.get_template(“time.html”)

c = Context({“today”:datetime.datetime.now(),”id”:p1,”name”:p2})

return HttpResponse(t.render(c))

urls.py:

urlpatterns = patterns(‘blog.views’,

url(r’^foo/(\d{4})/(\w+)/$’,’foo’), //或者下面一行的写法

url(r’^bar/(?P\d{4})/(?P\w+)/$’,’bar’), //这行的写法，固定了view函数中调用时候的参数名称

)

time.html:

此为模板页

在段输出相应字段即可

  • Today is {{ today | date:"Y-m-d" }}
