博客再次搬家

发表于 2015-05-22

看看文章归档信息，这份坚持已经从去年6月份坚持到现在。

努力争取每个月都有更新，也见证着自己的一点点的小进步。

从开始迷茫的不知所措，到现在笃定前行，需要太多努力。。。

从上个月开始发现SAE已经开始强制实名制，为了不给祖国添乱。

直接把博客程序搬下来，放到conoha上，速度不错。

反正也不指望博客赚钱，继续坚持下去吧。

希望明年的今天，我会成为我希望的那个大牛！

顺便记录下 这次搬家的历程和命令：

SAE：

1. 打包下载应用

2. 进mysql控制台，导出数据库文件xxx.sql

3. 把应用数据上传conoha主机

4. 导入sql：

mysql -u root -p

create database DB_NAME;

use DB_NAME;

source xxx.sql

quit

5. 修改/var/www/vhosts/blog/zb_users/c_option.php中的数据库连接字段

'ZC_MYSQL_SERVER' => 'localhost',
'ZC_MYSQL_USERNAME' => 'DN_USERNAME',
'ZC_MYSQL_PASSWORD' => 'PASSWORD',
'ZC_MYSQL_NAME' => DB_NAME,

6. 添加一个nginx的配置文件

针对不同的主机名头来访问： 

server {
listen 80;
server_name www.881027.com 881027.com;
#access_log logs/host.access.log main;
location / {
root /var/www/vhosts/blog;
index default.php index.php index.html index.htm;
#
#rewrite ^/(\w+)\.html$ /$1.php; 
#rewrite ^/(\w+)/(\w+)$ /$1/$2.php;
}
#
error_page 404 /error.html;
error_page 500 502 503 504 /50x.html;
location = /50x.html {
root html;
}
# pass the PHP scripts to FastCGI server listening on 127.0.0.1:9000
location ~ \.php$ {
root /var/www/vhosts/blog;
fastcgi_pass phpfpm;
fastcgi_index index.php;
fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
include fastcgi_params;
}
location ~ /\.ht {
deny all;
}
}

重启nginx

service nginx restart

修改www.881027.com，及*.881027.com
的A记录指定到新的ip

访问测试www.881027.com, 881027.com 都可以访问了。

成功！
